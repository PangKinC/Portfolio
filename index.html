About Me


Hi my name is Kin, and I'm a C# and ASP.NET Developer from London.

For a long time now I've always had a keen interest and passion for anything IT related. Which only continued to grow to the point that I dedicated my studies entirely to it throughout both college and university, taking the BTEC National Diploma in IT course and Computer Science with Games Techonology degree respectively.

Starting from college education onwards, I've gained knowledge of the various different subjects and factors in IT ranging from Human Computer Interface (HCI), e-Commerce, Networking and much more.

Being a logical thinker and both a problem analyzer and solver, from university onwards, I found that my drive leaned towards the programming side of things. Also during university was the time when I picked up my first real programming languages in Java and then eventually C++.

After learning these languages, I did both a second year Team Project as the lead programmer using Java, and my final year dissertation: a 2.5D platformer game using C++ with OpenGL. From graduation onwards, I took the Just IT course in order to further hone my programming skills and to have a portfolio completed in order secure a role as a Junior Back End Developer for the future.

Word Guessing Game

Overview


For this project, I created a simple word guessing game very similar to the popular game: Hangman. The objective of the game is to try get the highest score possible by the end after going through a set list of words.

In order to do this, players should aim to guess the word as quickly as possble for a higher score. Along with this there are two multipliers, one is for not getting any guesses incorrect, and the other multiplier being getting consecutive word guesses forming a chain.

I've recently also implemented a seperate page which details the objectives of the game along with some hint and tricks. On this page is also where a player can adjust the difficulty setting and choose the subject of the game.

Future Scope


  • Apply even more front-end programming (JavaScript / CSS / HTML) to make things more graphically appealing.
  • Create a database that logs the highest score gotten from the playerbase.

Languages / Tools Used


  • C#, ASP.NET Web Forms, JQuery, JavaScript, Bootstrap, HTML, CSS

Restaurant Finder

Overview


For my second project, this time I attempted to use ASP.NET with the MVC (Model View Controller) design pattern.
I used in conjuction with this a SQL database hand coded, with one table which stores a list of different restaurants and another table which takes in reviews for a single restaurant.

The main feature of this project is it has a action which by taking in the latitude and longitude of a address, a user can first search for his address location and if it's found it would then move to a page showing a Google Map with every restaurant (these are the same ones are that is stored inside the first database table) located nearby aforementioned address.

I've recently implemented a login user using the Identity 2.0 framework library, which i've expanded upon by creating User Roles (Admin and Guest). Using these roles, I was able to create authorization filters so that only specific roles can view specific pages.

Future Scope


  • Apply more front-end programming (CSS / HTML) to make things both more graphically appealing and interactive.
  • Flesh out the login system with even more features, templates automatically provided by Identity 2.0
  • Create a star-rating system to determine how popular a restaurant is by calculating the average.
